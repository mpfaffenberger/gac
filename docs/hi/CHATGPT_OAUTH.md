# GAC के साथ ChatGPT OAuth का उपयोग करना

**English** | [简体中文](../zh-CN/CHATGPT_OAUTH.md) | [繁體中文](../zh-TW/CHATGPT_OAUTH.md) | [日本語](../ja/CHATGPT_OAUTH.md) | [한국어](../ko/CHATGPT_OAUTH.md) | [हिन्दी](../hi/CHATGPT_OAUTH.md) | [Tiếng Việt](../vi/CHATGPT_OAUTH.md) | [Français](../fr/CHATGPT_OAUTH.md) | [Русский](../ru/CHATGPT_OAUTH.md) | [Español](../es/CHATGPT_OAUTH.md) | [Português](../pt/CHATGPT_OAUTH.md) | [Norsk](../no/CHATGPT_OAUTH.md) | [Svenska](../sv/CHATGPT_OAUTH.md) | [Deutsch](../de/CHATGPT_OAUTH.md) | [Nederlands](../nl/CHATGPT_OAUTH.md) | [Italiano](../it/CHATGPT_OAUTH.md)

GAC ChatGPT OAuth के माध्यम से प्रमाणीकरण का समर्थन करता है, जिससे आप OpenAI API कुंजियों के लिए अलग से भुगतान किए बिना अपनी ChatGPT सदस्यता का उपयोग करके OpenAI के Codex API तक पहुंच सकते हैं। यह OpenAI के Codex CLI द्वारा उपयोग की जाने वाली OAuth फ़्लो को दर्शाता है।

> ⚠️ **सावधानी — अनधिकृत उपयोग:** यह OpenAI के Codex CLI के समान OAuth फ़्लो का उपयोग करता है, और जबकि यह वर्तमान में काम करता है, OpenAI किसी भी समय तृतीय-पक्ष टोकन उपयोग को प्रतिबंधित कर सकता है। GAC काफी छोटा है कि यह अब तक रडार के नीचे उड़ गया है, लेकिन यहां ChatGPT OAuth का उपयोग तृतीय-पक्ष उपकरणों के लिए **आधिकारिक रूप से स्वीकृत नहीं** है और किसी भी समय काम करना बंद कर सकता है। यदि आपको विश्वसनीय कमिट संदेश जनरेशन की आवश्यकता है, तो सीधे API प्रदाता (`openai` आदि) का उपयोग करें। वर्तमान नीति के लिए [OpenAI का Codex दस्तावेज़ीकरण](https://openai.com/codex) देखें।

## ChatGPT OAuth क्या है?

ChatGPT OAuth आपको अपनी मौजूदा ChatGPT Plus या Pro सदस्यता का उपयोग करके कमिट संदेश उत्पन्न करने के लिए Codex API तक पहुंचने देता है। API कुंजियों और प्रति-टोकन बिलिंग को प्रबंधित करने के बजाय, आप अपने ब्राउज़र के माध्यम से एक बार प्रमाणित करते हैं और GAC टोकन जीवनचक्र को स्वचालित रूप से संभालता है।

## लाभ

- **लागत प्रभावी**: API एक्सेस के लिए अलग से भुगतान करने के बजाय अपनी मौजूदा ChatGPT Plus/Pro सदस्यता का उपयोग करें
- **समान मॉडल**: Codex-अनुकूलित मॉडल (`gpt-5.5`, `gpt-5.4`, `gpt-5.3-codex`) तक पहुंचें
- **कोई API कुंजी प्रबंधन नहीं**: ब्राउज़र-आधारित OAuth का अर्थ है कि कोई API कुंजियां घूमानी या स्टोर करने की आवश्यकता नहीं है
- **अलग बिलिंग**: ChatGPT OAuth उपयोग सीधे OpenAI API बिलिंग से अलग है

## सेटअप

GAC में ChatGPT के लिए अंतर्निहित OAuth प्रमाणीकरण शामिल है। सेटअप प्रक्रिया पूरी तरह से स्वचालित है और प्रमाणीकरण के लिए आपका ब्राउज़र खोलेगी।

### विकल्प 1: प्रारंभिक सेटअप के दौरान (अनुशंसित)

`uvx gac init` चलाते समय, बस अपने प्रदाता के रूप में "ChatGPT OAuth" चुनें:

```bash
gac init
```

विज़ार्ड करेगा:

1. आपको प्रदाता सूची से "ChatGPT OAuth" चुनने के लिए कहेगा
2. OAuth प्रमाणीकरण के लिए आपके ब्राउज़र को स्वचालित रूप से खोलेगा
3. आपके एक्सेस टोकन को `~/.gac/oauth/chatgpt-oauth.json` में सहेजेगा
4. डिफ़ॉल्ट मॉडल सेट करेगा

### विकल्प 2: बाद में ChatGPT OAuth पर स्विच करें

यदि आपके पास पहले से ही GAC को किसी अन्य प्रदाता के साथ कॉन्फ़िगर किया गया है और आप ChatGPT OAuth पर स्विच करना चाहते हैं:

```bash
gac model
```

फिर:

1. प्रदाता सूची से "ChatGPT OAuth" चुनें
2. आपका ब्राउज़र OAuth प्रमाणीकरण के लिए स्वचालित रूप से खुलेगा
3. टोकन `~/.gac/oauth/chatgpt-oauth.json` में सहेजा गया
4. मॉडल स्वचालित रूप से कॉन्फ़िगर किया गया

### सामान्य रूप से GAC का उपयोग करें

प्रमाणित होने के बाद, GAC का उपयोग हमेशा की तरह करें:

```bash
# अपने परिवर्तन स्टेज करें
git add .

# ChatGPT OAuth के साथ जनरेट और कमिट करें
gac

# या एकल कमिट के लिए मॉडल ओवरराइड करें
gac -m chatgpt-oauth:gpt-5.5
```

## उपलब्ध मॉडल

ChatGPT OAuth Codex-अनुकूलित मॉडल तक पहुंच प्रदान करता है। वर्तमान मॉडल में शामिल हैं:

- `gpt-5.5` — नवीनतम और सबसे सक्षम Codex मॉडल
- `gpt-5.4` — पिछली पीढ़ी का Codex मॉडल
- `gpt-5.3-codex` — तीसरी पीढ़ी का Codex मॉडल

उपलब्ध मॉडल की पूरी सूची के लिए [OpenAI दस्तावेज़ीकरण](https://platform.openai.com/docs/models) देखें।

## CLI कमांड

GAC ChatGPT OAuth प्रबंधन के लिए समर्पित CLI कमांड प्रदान करता है:

### लॉगिन

ChatGPT OAuth के साथ प्रमाणित या पुनः प्रमाणित करें:

```bash
gac auth chatgpt login
```

आपका ब्राउज़र OAuth फ़्लो पूरा करने के लिए स्वचालित रूप से खुलेगा। यदि आप पहले से प्रमाणित हैं, तो यह आपके टोकन को रीफ़्रेश करेगा।

### लॉगआउट

संग्रहीत ChatGPT OAuth टोकन निकालें:

```bash
gac auth chatgpt logout
```

यह `~/.gac/oauth/chatgpt-oauth.json` पर संग्रहीत टोकन फ़ाइल को हटा देता है।

### स्थिति

अपनी वर्तमान ChatGPT OAuth प्रमाणीकरण स्थिति जांचें:

```bash
gac auth chatgpt status
```

या सभी प्रदाताओं को एक साथ जांचें:

```bash
gac auth
```

## समस्या निवारण

### टोकन समाप्त हो गया

यदि आपको प्रमाणीकरण त्रुटियां दिखती हैं, तो आपका टोकन समाप्त हो सकता है। निम्नलिखित चलाकर पुनः प्रमाणित करें:

```bash
gac auth chatgpt login
```

आपका ब्राउज़र ताज़ा OAuth प्रमाणीकरण के लिए स्वचालित रूप से खुलेगा। GAC संभव होने पर पुनः प्रमाणन के बिना एक्सेस को नवीनीकृत करने के लिए रीफ़्रेश टोकन का स्वचालित रूप से उपयोग करता है।

### प्रमाणीकरण स्थिति जांचें

यह जांचने के लिए कि आप वर्तमान में प्रमाणित हैं या नहीं:

```bash
gac auth chatgpt status
```

या सभी प्रदाताओं को एक साथ जांचें:

```bash
gac auth
```

### लॉगआउट

अपने संग्रहीत टोकन को हटाने के लिए:

```bash
gac auth chatgpt logout
```

### "ChatGPT OAuth टोकन नहीं मिला"

इसका मतलब है कि GAC को आपका एक्सेस टोकन नहीं मिल रहा है। निम्नलिखित चलाकर प्रमाणित करें:

```bash
gac model
```

फिर प्रदाता सूची से "ChatGPT OAuth" चुनें। OAuth फ़्लो स्वचालित रूप से शुरू हो जाएगी।

### "प्रमाणीकरण विफल"

यदि OAuth प्रमाणीकरण विफल रहता है:

1. सुनिश्चित करें कि आपके पास सक्रिय ChatGPT Plus या Pro सदस्यता है
2. जांचें कि आपका ब्राउज़र सही ढंग से खुलता है
3. यदि समस्याएं बनी रहती हैं तो कोई अन्य ब्राउज़र आज़माएं
4. `auth.openai.com` से नेटवर्क कनेक्टिविटी सत्यापित करें
5. जांचें कि लोकल कॉलबैक सर्वर के लिए पोर्ट 1455-1465 उपलब्ध हैं

### पोर्ट पहले से ही उपयोग में है

OAuth कॉलबैक सर्वर स्वचालित रूप से पोर्ट 1455-1465 का प्रयास करता है। यदि सभी पोर्ट कब्जे में हैं:

```bash
# macOS/Linux पर:
lsof -ti:1455-1465 | xargs kill -9

# Windows (PowerShell) पर:
Get-NetTCPConnection -LocalPort 1455,1456,1457,1458,1459,1460,1461,1462,1463,1464,1465 | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
```

फिर `uvx gac auth chatgpt login` पुनः चलाएं।

## OpenAI प्रदाता से अंतर

| सुविधा       | OpenAI (`openai:`)                 | ChatGPT OAuth (`chatgpt-oauth:`)                              |
| ------------ | ---------------------------------- | ------------------------------------------------------------- |
| प्रमाणीकरण   | API कुंजी (`OPENAI_API_KEY`)       | OAuth (स्वचालित ब्राउज़र फ़्लो)                               |
| बिलिंग       | प्रति-टोकन API बिलिंग              | सदस्यता-आधारित (ChatGPT Plus/Pro)                             |
| सेटअप        | मैन्युअल API कुंजी प्रविष्टि       | `uvx gac init` या `uvx gac model` के माध्यम से स्वचालित OAuth |
| टोकन प्रबंधन | लंबे समय तक चलने वाली API कुंजियां | OAuth टोकन (रीफ़्रेश टोकन का उपयोग करके स्वचालित रीफ़्रेश)    |
| मॉडल         | सभी OpenAI मॉडल                    | Codex-अनुकूलित मॉडल                                           |

## सुरक्षा नोट्स

- **अपने एक्सेस टोकन को वर्जन कंट्रोल में कभी भी कमिट न करें**
- GAC OAuth टोकन को `~/.gac/oauth/chatgpt-oauth.json` में स्टोर करता है (आपके प्रोजेक्ट डायरेक्टरी के बाहर)
- OAuth फ़्लो बेहतर सुरक्षा के लिए PKCE (Proof Key for Code Exchange) का उपयोग करता है
- लोकल कॉलबैक सर्वर केवल localhost पर चलता है (पोर्ट 1455-1465)
- पुनः प्रमाणन के बिना एक्सेस को स्वचालित रूप से नवीनीकृत करने के लिए रीफ़्रेश टोकन का उपयोग किया जाता है
