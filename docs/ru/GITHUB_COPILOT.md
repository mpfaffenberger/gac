# Использование GitHub Copilot с GAC

GAC поддерживает аутентификацию через GitHub Copilot, позволяя вам использовать подписку Copilot для доступа к моделям от OpenAI, Anthropic, Google и других — всё это включено в ваш план GitHub Copilot.

## Что такое GitHub Copilot OAuth?

GitHub Copilot OAuth использует **Device Flow** — безопасный метод аутентификации через браузер, не требующий локального сервера обратного вызова. Вы переходите по URL, вводите одноразовый код и авторизуете GAC для использования вашего доступа Copilot. За кулисами GAC обменивает ваш долгосрочный GitHub OAuth-токен на краткосрочные токены сеанса Copilot (~30 мин), которые предоставляют доступ к API Copilot.

Это даёт вам доступ к моделям от нескольких поставщиков через одну подписку:

- **OpenAI** — `gpt-4o`, `gpt-4.1`, `gpt-4.1-mini`, `o3`, `o4-mini`
- **Anthropic** — `claude-opus-4.6`, `claude-opus-4`, `claude-sonnet-4`, `claude-sonnet-4.5`, `claude-haiku-4.5`
- **Google** — `gemini-2.5-pro`, `gemini-2.5-flash`

## Преимущества

- **Доступ к нескольким поставщикам**: Используйте модели от OpenAI, Anthropic и Google через одну подписку
- **Экономичность**: Используйте существующую подписку Copilot вместо отдельной оплаты ключей API
- **Без управления ключами API**: Аутентификация Device Flow — не нужно ротировать или хранить ключи
- **Поддержка GitHub Enterprise**: Работает с экземплярами GHE через флаг `--host`

## Настройка

### Вариант 1: Во время начальной настройки (Рекомендуется)

При запуске `uvx gac init` просто выберите «Copilot» в качестве вашего поставщика:

```bash
uvx gac init
```

Мастер:

1. Попросит вас выбрать «Copilot» из списка поставщиков
2. Отобразит одноразовый код и откроет ваш браузер для аутентификации Device Flow
3. Сохранит ваш OAuth-токен в `~/.gac/oauth/copilot.json`
4. Установит модель по умолчанию

### Вариант 2: Переключиться на Copilot позже

Если вы уже настроили GAC с другим поставщиком:

```bash
uvx gac model
```

Затем выберите «Copilot» из списка поставщиков и пройдите аутентификацию.

### Вариант 3: Прямой вход

Аутентифицируйтесь напрямую без изменения модели по умолчанию:

```bash
uvx gac auth copilot login
```

### Используйте GAC как обычно

После аутентификации используйте GAC как обычно:

```bash
# Индексируйте ваши изменения
git add .

# Генерируйте и коммитьте с Copilot
uvx gac

# Или переопределите модель для одного коммита
uvx gac -m copilot:gpt-4.1
uvx gac -m copilot:claude-sonnet-4.5
uvx gac -m copilot:gemini-2.5-pro
```

| --------- | ---------------------------------------------------------------------------------------------- |

## GitHub Enterprise

Для аутентификации на экземпляре GitHub Enterprise:

```bash
uvx gac auth copilot login --host ghe.mycompany.com
```

GAC автоматически использует правильные конечные точки Device Flow и API для вашего экземпляра GHE. Токен сеанса кэшируется для каждого хоста, поэтому разные экземпляры GHE обрабатываются независимо.

## Команды CLI

GAC предоставляет специализированные команды CLI для управления аутентификацией Copilot:

### Вход

Аутентифицируйтесь или повторите аутентификацию с GitHub Copilot:

```bash
uvx gac auth copilot login
```

Ваш браузер откроет страницу Device Flow, где вы введёте одноразовый код. Если вы уже аутентифицированы, будет предложено пройти повторную аутентификацию.

Для GitHub Enterprise:

```bash
uvx gac auth copilot login --host ghe.mycompany.com
```

### Выход

Удалите сохранённые токены Copilot:

```bash
uvx gac auth copilot logout
```

Это удаляет файл сохранённого токена по адресу `~/.gac/oauth/copilot.json` и кэш сеанса.

### Статус

Проверьте текущий статус аутентификации Copilot:

```bash
uvx gac auth copilot status
```

Или проверьте всех поставщиков сразу:

```bash
uvx gac auth
```

## Как это работает

Процесс аутентификации Copilot отличается от ChatGPT и Claude Code OAuth:

1. **Device Flow** — GAC запрашивает код устройства у GitHub и отображает его
2. **Авторизация в браузере** — Вы переходите по URL и вводите код
3. **Опрос токена** — GAC опрашивает GitHub, пока вы не завершите авторизацию
4. **Обмен токена сеанса** — GitHub OAuth-токен обменивается на краткосрочный токен сеанса Copilot
5. **Автообновление** — Токены сеанса (~30 мин) автоматически обновляются из кэшированного OAuth-токена

В отличие от OAuth на основе PKCE (ChatGPT/Claude Code), Device Flow не требует локального сервера обратного вызова или управления портами.

## Устранение неполадок

### «Аутентификация Copilot не найдена»

Выполните команду входа для аутентификации:

```bash
uvx gac auth copilot login
```

### «Не удалось получить токен сеанса Copilot»

Это означает, что GAC получил GitHub OAuth-токен, но не смог обменять его на токен сеанса Copilot. Обычно это означает:

1. **Нет подписки Copilot** — Ваша учётная запись GitHub не имеет активной подписки Copilot
2. **Токен отозван** — OAuth-токен был отозван; пройдите повторную аутентификацию с помощью `uvx gac auth copilot login`

### Токен сеанса истёк

Токены сеанса истекают примерно через 30 минут. GAC автоматически обновляет их из кэшированного OAuth-токена, поэтому вам не нужно часто проходить повторную аутентификацию. Если автообновление не удаётся:

```bash
uvx gac auth copilot login
```

### «Недопустимое или небезопасное имя хоста»

Флаг `--host` строго проверяет имена хостов для предотвращения SSRF-атак. Если вы видите эту ошибку:

- Убедитесь, что имя хоста не содержит портов (например, используйте `ghe.company.com`, а не `ghe.company.com:8080`)
- Не включайте протоколы или пути (например, используйте `ghe.company.com`, а не `https://ghe.company.com/api`)
- Частные IP-адреса и `localhost` заблокированы из соображений безопасности

### Проблемы с GitHub Enterprise

Если аутентификация GHE не удаётся:

1. Убедитесь, что на вашем экземпляре GHE включён Copilot
2. Проверьте, что имя хоста GHE доступно с вашего компьютера
3. Убедитесь, что ваша учётная запись GHE имеет лицензию Copilot
4. Попробуйте с явным указанием флага `--host`: `uvx gac auth copilot login --host ghe.mycompany.com`

## Отличия от других поставщиков OAuth

| Функция                 | ChatGPT OAuth                    | Claude Code                             | Copilot                                       |
| ----------------------- | -------------------------------- | --------------------------------------- | --------------------------------------------- |
| Метод аутентификации    | PKCE (обратный вызов браузера)   | PKCE (обратный вызов браузера)          | Device Flow (одноразовый код)                 |
| Сервер обратного вызова | Порты 1455-1465                  | Порты 8765-8795                         | Не нужен                                      |
| Время жизни токена      | Долгосрочные (автообновление)    | С истечением (повторная аутентификация) | Сеанс ~30 мин (автообновление)                |
| Модели                  | Оптимизированные Codex от OpenAI | Семейство Claude                        | Мультипоставщик (OpenAI + Anthropic + Google) |
| Поддержка GHE           | Нет                              | Нет                                     | Да (флаг `--host`)                            |

## Примечания по безопасности

- **Никогда не коммитьте ваш OAuth-токен** в систему контроля версий
- GAC хранит OAuth-токены в `~/.gac/oauth/copilot.json` (за пределами каталога вашего проекта)
- Токены сеанса кэшируются в `~/.gac/oauth/copilot_session.json` с правами доступа `0o600`
- Имена хостов строго проверяются для предотвращения SSRF и URL-инъекций
- Частные IP-адреса, адреса loopback и `localhost` заблокированы как имена хостов
- Device Flow не открывает локальные порты, что уменьшает поверхность атаки
