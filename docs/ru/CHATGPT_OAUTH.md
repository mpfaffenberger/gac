# Использование ChatGPT OAuth с GAC

GAC поддерживает аутентификацию через ChatGPT OAuth, позволяя вам использовать вашу подписку ChatGPT для доступа к API Codex от OpenAI вместо отдельной оплаты ключей API OpenAI. Это отражает тот же поток OAuth, используемый Codex CLI от OpenAI.

> ⚠️ **Внимание — несанкционированное использование:** Это использует тот же поток OAuth, что и Codex CLI от OpenAI, и хотя в настоящее время это работает, OpenAI может в любое время ограничить использование токенов третьими сторонами. GAC достаточно мал, чтобы до сих пор оставаться незамеченным, но использование ChatGPT OAuth здесь **официально не одобрено** для сторонних инструментов и может перестать работать в любое время. Если вам нужна надежная генерация сообщений коммитов, используйте прямого поставщика API (`openai` и т. д.). См. [документацию Codex от OpenAI](https://openai.com/codex) для получения информации о текущей политике.

## Что такое ChatGPT OAuth?

ChatGPT OAuth позволяет вам использовать существующую подписку ChatGPT Plus или Pro для доступа к API Codex для генерации сообщений коммитов. Вместо управления ключами API и оплатой за токены вы аутентифицируетесь один раз через браузер, и GAC автоматически обрабатывает жизненный цикл токена.

## Преимущества

- **Экономичность**: Используйте существующую подписку ChatGPT Plus/Pro вместо отдельной оплаты доступа к API
- **Те же модели**: Доступ к оптимизированным для Codex моделям (`gpt-5.5`, `gpt-5.4`, `gpt-5.3-codex`)
- **Без управления ключами API**: OAuth на основе браузера означает, что не нужно ротировать или хранить ключи API
- **Отдельная оплата**: Использование ChatGPT OAuth отделено от прямой оплаты API OpenAI

## Настройка

GAC включает встроенную аутентификацию OAuth для ChatGPT. Процесс настройки полностью автоматизирован и откроет ваш браузер для аутентификации.

### Вариант 1: Во время начальной настройки (Рекомендуется)

При запуске `uvx gac init` просто выберите «ChatGPT OAuth» в качестве вашего поставщика:

```bash
uvx gac init
```

Мастер:

1. Попросит вас выбрать «ChatGPT OAuth» из списка поставщиков
2. Автоматически откроет ваш браузер для аутентификации OAuth
3. Сохранит ваш токен доступа в `~/.gac/oauth/chatgpt-oauth.json`
4. Установит модель по умолчанию

### Вариант 2: Переключиться на ChatGPT OAuth позже

Если вы уже настроили GAC с другим поставщиком и хотите переключиться на ChatGPT OAuth:

```bash
uvx gac model
```

Затем:

1. Выберите «ChatGPT OAuth» из списка поставщиков
2. Ваш браузер автоматически откроется для аутентификации OAuth
3. Токен сохранен в `~/.gac/oauth/chatgpt-oauth.json`
4. Модель настроена автоматически

### Используйте GAC как обычно

После аутентификации используйте GAC как обычно:

```bash
# Индексируйте ваши изменения
git add .

# Генерируйте и коммитьте с ChatGPT OAuth
uvx gac

# Или переопределите модель для одного коммита
uvx gac -m chatgpt-oauth:gpt-5.5
```

## Доступные модели

ChatGPT OAuth предоставляет доступ к оптимизированным для Codex моделям. Текущие модели включают:

- `gpt-5.5` — Последняя и самая мощная модель Codex
- `gpt-5.4` — Модель Codex предыдущего поколения
- `gpt-5.3-codex` — Модель Codex третьего поколения

Проверьте [документацию OpenAI](https://platform.openai.com/docs/models) для получения полного списка доступных моделей.

## Команды CLI

GAC предоставляет специализированные команды CLI для управления ChatGPT OAuth:

### Вход

Аутентифицируйтесь или повторите аутентификацию с ChatGPT OAuth:

```bash
uvx gac auth chatgpt login
```

Ваш браузер автоматически откроется для завершения потока OAuth. Если вы уже аутентифицированы, это обновит ваши токены.

### Выход

Удалите сохраненные токены ChatGPT OAuth:

```bash
uvx gac auth chatgpt logout
```

Это удаляет файл сохраненного токена по адресу `~/.gac/oauth/chatgpt-oauth.json`.

### Статус

Проверьте текущий статус аутентификации ChatGPT OAuth:

```bash
uvx gac auth chatgpt status
```

Или проверьте всех поставщиков сразу:

```bash
uvx gac auth
```

## Устранение неполадок

### Токен истек

Если вы видите ошибки аутентификации, ваш токен мог истечь. Повторите аутентификацию, выполнив:

```bash
uvx gac auth chatgpt login
```

Ваш браузер автоматически откроется для новой аутентификации OAuth. GAC автоматически использует токены обновления для продления доступа без повторной аутентификации, когда это возможно.

### Проверить статус аутентификации

Чтобы проверить, аутентифицированы ли вы в настоящее время:

```bash
uvx gac auth chatgpt status
```

Или проверьте всех поставщиков сразу:

```bash
uvx gac auth
```

### Выход

Чтобы удалить сохраненный токен:

```bash
uvx gac auth chatgpt logout
```

### «Токен ChatGPT OAuth не найден»

Это означает, что GAC не может найти ваш токен доступа. Аутентифицируйтесь, выполнив:

```bash
uvx gac model
```

Затем выберите «ChatGPT OAuth» из списка поставщиков. Поток OAuth запустится автоматически.

### «Аутентификация не удалась»

Если аутентификация OAuth не удалась:

1. Убедитесь, что у вас есть активная подписка ChatGPT Plus или Pro
2. Проверьте, правильно ли открывается ваш браузер
3. Попробуйте другой браузер, если проблемы сохраняются
4. Проверьте подключение к сети к `auth.openai.com`
5. Проверьте, что порты 1455-1465 доступны для локального сервера обратного вызова

### Порт уже используется

Сервер обратного вызова OAuth автоматически пытается использовать порты 1455-1465. Если все порты заняты:

```bash
# На macOS/Linux:
lsof -ti:1455-1465 | xargs kill -9

# На Windows (PowerShell):
Get-NetTCPConnection -LocalPort 1455,1456,1457,1458,1459,1460,1461,1462,1463,1464,1465 | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
```

Затем повторно выполните `uvx gac auth chatgpt login`.

## Отличия от поставщика OpenAI

| Функция             | OpenAI (`openai:`)          | ChatGPT OAuth (`chatgpt-oauth:`)                                             |
| ------------------- | --------------------------- | ---------------------------------------------------------------------------- |
| Аутентификация      | Ключ API (`OPENAI_API_KEY`) | OAuth (автоматизированный поток браузера)                                    |
| Оплата              | Оплата API за токены        | На основе подписки (ChatGPT Plus/Pro)                                        |
| Настройка           | Ручной ввод ключа API       | Автоматический OAuth через `uvx gac init` или `uvx gac model`                |
| Управление токенами | Долговечные ключи API       | Токены OAuth (автоматическое обновление с использованием токенов обновления) |
| Модели              | Все модели OpenAI           | Оптимизированные для Codex модели                                            |

## Примечания по безопасности

- **Никогда не коммитьте ваш токен доступа** в систему контроля версий
- GAC хранит токены OAuth в `~/.gac/oauth/chatgpt-oauth.json` (за пределами каталога вашего проекта)
- Поток OAuth использует PKCE (Proof Key for Code Exchange) для повышенной безопасности
- Локальный сервер обратного вызова работает только на localhost (порты 1455-1465)
- Токены обновления используются для автоматического продления доступа без повторной аутентификации
